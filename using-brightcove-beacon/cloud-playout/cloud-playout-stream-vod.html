Cloud Playout 모듈을 사용하면 방송사와 미디어 게시자가 VOD 비디오 및 라이브 스트림 모음을 사용한 다음 선형 채널로 스트리밍할 수 있습니다. 채널은 Brightcove Live, Amazon S3 또는 지정된 RTMP URL로 스트리밍할 수 있습니다.

Cloud Playout 모듈에 액세스하려면클라우드 플레이아웃탐색 메뉴에서

생성된 모든 Cloud Playout 채널 목록이 표시됩니다(있는 경우).

페이지에 다음 정보가 표시됩니다.

  • 채널 이름 - 채널 이름
  • 시작 시간 - 채널 시작 시간
  • 종료 시간 - 채널 종료 시간
  • 루핑 - 프로그램이 반복되는지 여부를 나타냅니다.
  • 목적지 - 채널의 대상 목적지
  • 생성 일자 - 채널이 생성된 날짜
  • 마지막 업데이트 - 채널이 마지막으로 업데이트된 날짜
  • 상태 - 채널 상태
    • 초안 - 채널이 생성되었지만 활성화되지 않았습니다.
    • 만들기 - 채널이 활성화되는 중입니다.
    • 예정 - 채널이 활성화되어 스트리밍할 준비가 되었습니다.
    • 시작 중 - 스트리밍을 시작하기 위한 채널 시작 프로세스
    • 달리기 - 채널이 활발하게 스트리밍 중입니다.
    • 멎는 - 채널 스트리밍이 중지됩니다.
    • 게으른 - 채널이 성공적으로 실행되고 중지되었습니다.
    • 삭제 중 - 채널 삭제 중
    • 오류 - 채널 생성 또는 스트리밍 중 오류가 발생했습니다.

채널 만들기

새 Cloud Playout 채널을 만들려면 다음 단계를 따르세요.

  1. 딸깍 하는 소리클라우드 플레이아웃탐색 헤더에서.
  2. 딸깍 하는 소리 :
  3. 채널 입력신분증세부:
    • 이름 - 채널의 이름입니다. 이름에는 영문자가 포함되어야 하며 다음과 같은 특수 문자가 허용됩니다. _ . : / = + - 
    • 설명 - 채널 설명
    • 이미지 URL - 채널의 이미지 파일 URL 채널 메타데이터의 일부로 검색할 수 있는 채널 이미지를 의미합니다.
  4. 채널 입력일정세부.
    • 루프 재생 콘텐츠 - 만약에에 , 프로그램은 루프에서 재생되고종료 날짜/시간활성화됩니다. 프로그램은 언제든지 수정할 수 있으며 변경 사항은 현재 프로그램과 다음 루프에 반영됩니다. 참조루핑 프로그램 만들기아래 섹션.
    • 광고 삽입 - 켜져 있으면 채널 재생 중에 광고를 표시할 수 있습니다. 이 옵션을 선택하면 라이브 입력이 허용되지 않고 출력만 가능합니다. 브라이트코브 라이브 이벤트또는 Amazon S3 버킷 . 보다콘텐츠 수익 창출 .
    • 시작 날짜/시간 - 채널이 재생되기 시작하는 시간입니다. 모든 자산의 시작 시간은 이 시작 시간에 지속 시간을 더하여 계산됩니다. 이 시간은 채널이 실행 상태가 아닐 때만 업데이트할 수 있습니다.
    • 종료 날짜/시간 - 값이 입력되지 않은 경우루프 재생 콘텐츠선택하지 않으면 재생이 완료되면 스트림이 종료됩니다. 만약에루프 재생 콘텐츠가 선택되고 없습니다. 종료 날짜/시간 , 재생은 채널이 중지될 때까지 스트리밍됩니다. 이 시간은 언제든지 업데이트할 수 있습니다.
  5. 라이브 입력을 사용할 경우 다음을 설정하십시오. 라이브 입력기본 Brightcove MP4를 슬레이트에 사용하지 않는 한 Slate URL을 입력합니다. 인코더가 중지되었거나 일부 기술적인 문제로 인해 라이브 스트림이 차단되어 라이브 입력이 중지될 때마다 슬레이트 MP4가 재생됩니다. 참고 Slate URL은 항상 사용할 수 있는 정적 URL이어야 합니다. (단기 CDN URL은 허용되지 않습니다.)
  6. 채널 입력목적지세부. Cloud Playout은 동시에 2개의 출력을 지원합니다. 예를 들어 HLS를 S3 버킷으로 출력하고 RTMP 피드를 Brightcove 라이브 채널 또는 타사 수집 지점으로 출력할 수 있습니다. 참고하세요. 브라이트코브 라이브 이벤트그리고 RTMP동시 대상은 지원되지 않습니다.
    • 디스플레이 최적화 - 출력의 비트 전송률 및 크기를 제어합니다.
      • 모바일 및 웹 - 체크 시 HD 출력(RTMP 또는 HLS 단일 비트 전송률)
      • 스마트 티비 - 체크 시 Full HD 출력(RTMP 또는 HLS 단일 비트 전송률)
    • 출력 영역 - 채널이 생성될 지역입니다. Brightcove는 대부분의 청중이 예상되는 위치와 지리적으로 가장 가까운 지역을 선택할 것을 권장합니다.
    • 출력 - 채널의 출력 방식 선택
      • 브라이트코브 라이브 이벤트 - 이전에 만든 Brightcove 라이브 채널을 만들거나 사용합니다. 새 채널을 만들 때 채널의 영역은 다음을 기반으로 최적화됩니다. 출력 영역선택한 것입니다.
      • RTMP - 스트림을 RTMP 또는 RTP 수집 지점으로 출력하려면 선택합니다. RTP는 타사 수집 지점에 사용할 수 없습니다. 때만 사용할 수 있습니다. 광고 삽입그리고브라이트코브 라이브 이벤트사용 가능. 선택하면 RTMP URL필드가 나타납니다 - 라이브 스트리밍 서비스의 RTMP 입력 URL을 입력하십시오.
      • Amazon S3 버킷 - HLS 출력을 Amazon S3 버킷으로 푸시하려면 선택합니다. 버킷 이름, 지역, 액세스 키 및 비밀 키를 입력해야 합니다. 버킷은 공개되고 쓰기 가능해야 합니다.
      로 출력한다는 점에 유의하십시오. 브라이트코브 라이브 이벤트청구 가능한 계정에 대한 실시간 시간을 자동으로 생성합니다. Cloud Playout 사용에 대한 추가 요금은 없지만 라이브 스트리밍 시간에 대한 일반 요금이 적용됩니다. RTMP 또는 Amazon S3로 출력하면 청구 가능한 라이브 시간이 생성되지 않습니다.
  7. 딸깍 하는 소리 . 상태로 새 채널이 생성됩니다. 초안 .

채널에 콘텐츠 추가

보다 Cloud Playout 채널에 콘텐츠 추가채널에 VOD 콘텐츠, 재생 목록 및 라이브 이벤트를 추가하는 방법에 대한 정보를 확인하세요.

또한 참조콘텐츠에 범퍼 추가 VOD 또는 라이브 이벤트 항목에 범퍼를 추가합니다.

비디오 및 라이브 이벤트가 채널에 추가된 후활성화채널을 활성화합니다. 스트리밍을 시작하려면 먼저 채널을 활성화해야 합니다. 참조채널 작업 사용자세한 내용은.

메모: 프로그램에서 비디오의 순서를 변경하려면 비디오를 클릭하고 새 위치로 끕니다.

비디오가 채널에 추가되면 시스템 생성 재생 목록이 만들어지고 미디어 모듈에서 볼 수 있습니다.

Cloud Playout 모듈에서 채널을 변경하는 것이 좋습니다. 어떤 식으로든 생성된 재생 목록을 편집하지 마십시오. .

비콘 채널에 대한 정보 얻기

Video Cloud를 떠나 Beacon으로 이동하기 전에 Beacon 채널을 설정하는 데 필요한 정보를 캡처해야 합니다. 보다비콘 채널 생성 Cloud Playout 채널에 대한 정보가 입력되는 방식을 확인합니다.

  1. 채널 목록에서 채널을 클릭하여 채널 세부 정보 페이지로 이동합니다.
  2. 아직 하지 않았다면 다음을 클릭하십시오. 활성화채널을 활성화합니다. 보다채널 작업 사용자세한 사항은.
  3. 채널이 활성화되면 기본 탐색을 사용하여미디어기준 치수.
  4. 비디오 목록 상단에 Cloud Playout 채널용으로 생성된 새 라이브 비디오가 표시되어야 합니다.
  5. 둘 다 저장비디오 ID그리고채널 ID (뒤에 오는 긴 영숫자 문자열클라우드 플레이아웃:태그에서).

채널 ID는 Cloud Playout 모듈의 채널 세부정보에서도 사용할 수 있습니다.
