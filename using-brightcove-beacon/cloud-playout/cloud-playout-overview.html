소개

Brightcove Cloud Playout은 라이브 피드와 결합된 VOD 카탈로그 항목의 예정된 재생을 프로그래밍하고 Brightcove Live의 안정성과 확장성을 모두 갖춘 선형 채널로 스트리밍할 수 있는 Video Cloud Studio 모듈입니다.

그런 다음 생성할 때 관련 정보를 입력하여 Cloud Playout 채널을 비콘 채널로 사용할 수 있습니다. 비콘 채널 . Cloud Playout 채널을 사용할 때의 주요 이점 중 하나는 EPG 서비스가 Cloud Playout에 내장되어 있기 때문에 타사 EPG 공급자를 찾아 통합할 필요가 없다는 것입니다.

Cloud Playout은 다음을 포함하여 예정된 프로그래밍 채널을 위한 강력한 종단 간 솔루션을 제공합니다.

  • 24x7 선형 채널
  • 팝업 / 가상 채널
  • 시뮬레이션된 라이브 이벤트

Cloud Playout은 사전 녹화된 콘텐츠를 활용하여 방송과 같은 라이브 TV 경험을 통해 청중 참여를 늘리고 새로운 수익 기회를 창출하는 데 도움이 됩니다. 기존 VOD 자산을 사용하여 예약된 라이브 채널을 만듭니다.

특징

Cloud Playout의 기능은 다음과 같습니다.

  • Video Cloud 미디어 라이브러리에서 VOD 자산을 가져옵니다.
  • VOD 자산과 예약된 라이브 피드 간에 전환합니다.
  • 쉽고 빠르고 안정적인 방식으로 막바지 일정 변경에 실시간으로 대응합니다.
  • SSAI(서버 측 광고 삽입)로 채널 콘텐츠에서 수익을 창출하세요.
  • 프로그램을 반복하도록 채널을 구성합니다.
  • 장치별 재생을 최적화합니다.
  • 실행 상태에서도 언제든지 드래그 앤 드롭하여 프로그램 일정을 재정렬할 수 있습니다.
  • 채널에 대한 EPG(전자 프로그램 가이드)를 검색하는 API

워크플로 개요

Cloud Playout 채널을 만들고 Beacon 앱에 추가하는 데는 두 부분이 있습니다.

  1. 에비디오 클라우드 , 채널을 만듭니다.
  2. 에봉홧불 , Cloud Playout 채널을 소스로 사용하는 새 비콘 채널을 만듭니다.

제한 사항

다음 가정 및 제한 사항이 적용됩니다.

  • 동영상은 Dynamic Delivery 프로필을 사용하여 수집되거나 다시 코드 변환되어야 합니다. MP4 버전이 있어야 합니다.
  • Cloud Playout에 지원되는 최대 입력 비트 전송률 및 해상도는 20MBPS/1080p입니다. 이는 VOD 콘텐츠와 라이브 입력 모두에 적용됩니다.
  • Brightcove는 비디오와 전자 프로그램 가이드 사이의 올바른 전환을 보장하기 위해 최소 2분 길이의 비디오를 사용할 것을 권장합니다.
  • 현재 동영상에 추가된 캡션(텍스트 트랙)은 Cloud Playout 채널에 표시되지 않습니다. 캡션 지원은 향후 릴리스에서 추가될 예정입니다.
  • 라이브 이벤트의 최소 길이는 5분입니다.
  • Cloud Playout의 모든 라이브 이벤트는 Video Cloud 원격 자산과 1:1 관계를 공유합니다. 이 원격 자산은 다른 채널에서 재사용할 수 없습니다. 라이브 이벤트가 채널에서 제거되면 연결된 원격 자산이 자동으로 삭제됩니다. 채널이 삭제되면 연결된 원격 자산도 자동으로 삭제됩니다.
  • 비디오공유다른 계정에서 계정으로 현재 Cloud Playout 채널에 추가할 수 없습니다.
  • 라이브 이벤트가 현재 재생 중이지만 사용 가능한 활성 피드가 없는 경우 라이브 피드가 재개되거나 이벤트가 종료될 때까지 슬레이트 콘텐츠가 루프에서 재생됩니다. 슬레이트 MP4 재생이 시작되기까지 약 30초의 지연이 있습니다.
  • Video Cloud 계정이 지리적 제한에 대해 구성된 경우 채널출력 영역 Video Cloud 지역 제한 정책과 호환되는 지역에서 생성해야 합니다.
  • 최상의 경험을 위해 Brightcove는 Google Chrome 브라우저 사용을 권장합니다.
  • SSAI 지원 채널의 경우 대상을 광고 지원 RTP 라이브 이벤트로 설정해야 합니다.
