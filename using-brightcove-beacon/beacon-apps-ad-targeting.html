소개

클라이언트 측 정보는 앱에서 SSAI로 전달되고 SSAI는 이 데이터를 고객의 VAST 태그에 포함합니다. 그러면 고객이 이 데이터를 기반으로 광고를 타겟팅할 수 있습니다. 이것의 목적은 고객의 광고 인벤토리의 가치를 높여 비즈니스의 광고 수익을 높이는 것입니다.

이 데이터가 광고 서버 또는 광고 제공업체에 전달되는 방식을 설정할 수 있습니다.

혜택

향상된 광고 타겟팅

  • 사용자 세션 ID:사용자가 앱을 열 때 각 기기에서 세션 ID를 생성합니다. 이를 통해 해당 세션에 광고 게재빈도를 설정하고 경쟁 광고가 함께 게재되지 않도록 제외하고 해당 세션에서 광고를 순차적으로 순환게재할 수 있습니다. 이러한 타겟팅 기능은 직접 판매 및 프로그래매틱 광고에 모두 필요합니다.
  • 프로그래매틱 광고에 대한 추가 데이터: 모바일 앱과 CTV 앱은 웹에서 사용자/기기 기반 광고 타겟팅을 위한 지배적인 방법인 쿠키를 지원하지 않습니다. 장치 ID와 같은 매개변수를 포함하면 이 귀중한 정보를 프로그래밍 방식 광고 구매자에게 전달할 수 있으며, 그러면 광고 타겟팅 기능이 향상되어 CPM이 향상됩니다.

광고 인벤토리 분할 지원

Amazon, Roku 및 Samsung(미국)에서 실행되는 앱의 경우 플랫폼에서 광고 인벤토리/수익을 70/30으로 분할해야 합니다. Beacon 고객은 광고 인벤토리의 70%를 수익화하고 플랫폼은 광고 인벤토리의 나머지 30%를 얻습니다. 이 향상된 광고 타겟팅을 사용하면 모든 앱에서 하나의 광고 태그를 사용하고 기기 매개변수를 광고 서버에 전달할 수 있습니다. 광고 서버 내에서 기기별로 광고 인벤토리를 분할하는 비즈니스 로직을 구현합니다. 플랫폼은 또한 광고 인벤토리 분할을 위해 LAT(Limited Ad Targeting)를 전달해야 합니다. 이 매개변수는 현재 광고 서버로 보내는 매개변수입니다.

사용자 개인 정보 개선

  • LAT "제한된 광고 타겟팅" - Beacon 앱 사용자가 사용자 행동을 기반으로 광고 타겟팅을 선택 해제할 수 있습니다.
  • 사용자는 언제든지 장치에 할당한 고유한 장치 ID를 재설정할 수 있습니다.

노트

  • 이 기능은 모든 플랫폼/기기 iOS, Apple TV, Android, Roku, Fire TV, 웹 및 스마트 TV에서 작동합니다.
  • 비콘 앱 광고 타겟팅은 VOD SSAI 및 라이브 SSAI 모두와 함께 사용할 수 있습니다.

VOD 단계

비디오 클라우드 파트 - VOD

  1. 다음에서 광고 구성 만들기 Studio의 서버측 광고 설정 페이지 :
    1. 딸깍 하는 소리광고 구성 만들기그리고 선택 VOD
    2. 광고 구성 이름 추가<./li>
    3. 광고 응답을 선택하세요. VMAP 또는 Vast가 권장됩니다. Vast를 사용하는 경우 광고 시간이 표시되어야 하는 위치를 지정하기 위해 비디오에 큐 포인트를 추가해야 합니다.
    4. 광고 태그를 붙여넣습니다(광고 제공업체에서 제공).
    5. Beacon 광고 구성에 필요하므로 광고 구성 ID를 복사합니다.

비콘 파트 - VOD

  1. Beacon Classic UI를 열고 광고 탭으로 이동합니다.
    1. 딸깍 하는 소리새 구성 추가
    2. 광고 구성 이름 지정
    3. 선택하다브라이트코브 SSAI
    4. 붙여넣기 SSAI 광고 구성 ID ~ 안으로 VOD 광고 구성 ID
  2. Studio 및 Beacon Classic에서 광고 구성을 설정했으면 고객 광고 서버에 전달할 클라이언트 측 매개변수를 구현해야 합니다. 클라이언트 측 매개변수는 재생 API(VMAP/매니페스트 진입점에서 제공됨)에서 전달할 수 있습니다. 이러한 매개변수는 SSAI 인프라에서 다음 형식으로 사용할 수 있습니다. {{url.xxx}} xxx가 재생 API URL에 제출된 매개변수 키에 해당하는 매크로.
    1. 에 광고 타겟팅 매개변수를 입력합니다. 광고 타겟비콘의 필드입니다. 입력은 쿼리 형식의 문자열입니다. 예시: d= {PLATFORM} &os= {OS} &남자= {MFGR}월(&mo)= {MODEL}앱피드(&appid)= {APP_ID} &lan= {LOCALE} &did= {RDID} &dt= {IDTYPE} &lat= {IS_LAT} &sid= {SID}
    2. 등호 앞에 키를 구성할 수 있습니다. . 그들해야하다 SSAI 광고 태그에 추가된 키와 일치합니다.
    3. 다음으로 광고 태그를 업데이트하십시오. SSAI 스튜디오 설정페이지. 다음은 Beacon Classic Ad-Targeting에 입력되고 SSAI Ad 태그에 추가된 매개변수의 예입니다. d= {PLATFORM} &os= {OS}앱피드(&appid)= {APP_ID} &did= {RDID} &dt= {IDTYPE} &lat= {IS_LAT} &sid= {SID}
    4. 이 매개변수는 이 광고 태그에서 다음과 같이 번역됩니다. https://ads.brightcove.com/ads?tech=dfpadrules&dur=15&pre=3&d= {{url.d}} &os= {{url.os}}앱피드(&appid)= {{url.appid}} &did= {{url.did}} &dt= {{url.dt}} &lat= {{url.latl}} &sid= {{url.sid}}

라이브 단계

  1. 먼저 SSAI 및 Beacon 구성 모두에서 필요하므로 광고 타겟팅 매개변수 문자열을 구성합니다. VOD용 매개변수를 재사용하거나 다른 세트를 선택할 수 있습니다.

    예시:

    아이피= {{client.ipaddress}}숫자(&N)= {{random.int32}} &d= {{PLATFORM}} &os= {{OS}} &did= {{RDID}} &dt= {{IDTYPE}} &lat= {{IS_LAT}}
  2. 다음에서 광고 구성 만들기 Studio의 서버 측 광고 설정 페이지 :
    1. 딸깍 하는 소리광고 구성 만들기그리고 선택살다
    2. 광고 구성의 이름 추가
    3. 선택하다대단한귀하의 광고 응답
    4. 광고 태그에 붙여넣고광고 타겟팅 매개변수의 쿼리 문자열 추가 .

      예시:

      https://ad-provider-host.com/path/to/ad-handler?ip= {{client.ipaddress}}숫자(&N)= {{random.int32}} &d= {{PLATFORM}} &os= {{OS}} &did= {{RDID}} &dt= {{IDTYPE}} &lat= {{IS_LAT}}
    5. 비콘 광고 구성에 대한 광고 구성 ID 복사
  3. Beacon Classic UI를 열고 광고 탭으로 이동합니다.
    1. 딸깍 하는 소리새 구성 추가
    2. 광고 구성 이름 지정
    3. 선택하다브라이트코브 SSAI
    4. SSAI 광고 구성 ID를라이브 광고 구성 ID
  4. 광고 타겟팅 매개변수 문자열을 붙여넣습니다. 광고 타겟들.

    예시:
