    라이브 이벤트 스트리밍

    이 항목에서는 라이브 이벤트를 브라이트코브 비콘 앱으로 스트리밍하는 방법을 배웁니다.

    개요

    라이브 이벤트를 애플리케이션으로 스트리밍하려면 일련의 단계를 수행하여 Video Cloud Live 계정과 브라이트코브 비콘을 연결해야 합니다.

    프로세스의 상위 수준 뷰는 다음과 같습니다.

    1. 브라이트코브 비콘에서 이벤트와 재생 목록을 만듭니다.
    2. Video Cloud에서 라이브 모듈에 액세스하여 이벤트를 생성하고 작업 ID를 얻습니다.
    3. 브라이트코브 비콘에서 Video Cloud의 작업 ID를 사용하여 라이브 이벤트를 등록합니다.
    4. 라이브 동영상은 브라이트코브 비콘 앱에서 스트리밍을 시작합니다.

    이 문서의 마지막 섹션에서는 정적 진입점을 사용하여 라이브에서 다른 유형의 이벤트를 만드는 단계를 볼 수 있습니다. 이를 통해 반복 라이브 이벤트를 활성화 및 비활성화할 수 있으며 한 번만 구성하면 됩니다.

    이 문서의 나머지 부분에서는 위의 각 단계에 대한 세부 정보를 제공합니다.

    브라이트코브 비콘에서 라이브 이벤트 만들기

    브라이트코브 비콘에서 이벤트를 생성하려면 다음 단계를 따르세요.

    브라이트코브 비콘 라이브로활성화된 계정만 탭 섹션에서재생 목록및이벤트를사용할 수 있습니다. 활성화되지 않은 경우 브라이트코브 지원에 문의하십시오.
    1. 브라이트코브 비콘에서이벤트탭을 클릭하여 이벤트 설정을 시작합니다.
    2. 새 이벤트 추가를클릭합니다.
    3. 라이브 이벤트에이름과이벤트 시작 날짜를지정합니다.
    4. 텍스트 데이터탭에서 이벤트 정보를 입력합니다.
    5. 비 텍스트 데이터탭을 클릭합니다.
    6. 아래 요구 사항에 맞는인물 사진 포스터및가로 포스터의이미지를 제공하십시오. 썸네일및와이드 배너에대한 이미지를 제공 할 필요가 없습니다.
      이미지 요구 사항
      초상화 포스터 포스터 이미지 (2:3)
      최소 너비: 1000픽셀 높이: 563픽셀
      최대 너비: 4000픽셀 높이: 2250픽셀
      풍경 포스터 가로 이미지 (16:9)
      최소 너비: 1000픽셀 높이: 563픽셀
      최대 너비: 4000픽셀 높이: 2250픽셀
    7. 다음과 같이권한 및 일정탭을 클릭합니다.
    8. 다음 필드에 값을 추가합니다.
      • 권한 유형 : 광고 , 구독또는무료중에서 권리 유형을 선택합니다.
      • 시작 시간 : 이벤트 시작 날짜를 설정합니다.
      • 종료 시간 : 이벤트의 종료 날짜를 설정합니다.
      • 최대 스트림 : 라이브 이벤트를 볼 수 있는 최대 스트림 수를 설정합니다. 기본 숫자는 0 (영) 입니다.
      • 게시할 디바이스 : 게시할 장치를 추가합니다. 모든 장치 선택/선택 취소를 클릭하여 사용 가능한 장치 목록을 확인합니다.
      • 허용된 위치 : World로들어가는 제한이 없는 경우 콘텐츠를 스트리밍할 위치를 선택합니다.
      • 거부된 위치 : 콘텐츠를 스트리밍하지 않을 위치를 선택합니다.
    9. 새 이벤트 만들기버튼을 클릭하여 이벤트를 만듭니다.

    재생 목록에 브라이트코브 비콘 이벤트 추가

    이벤트를 만들고 나면 다음 작업은 이벤트를 재생 목록에 추가하는 것입니다. 다음 단계는 해당 프로세스를 안내합니다.

    1. 브라이트코브 비콘에서재생 목록탭을 클릭합니다.
    2. 새 재생 목록 추가버튼을 클릭합니다.
      브라이트코브 비콘에서 라이브 이벤트를 포함할 재생 목록을 만들어야 합니다. Video Cloud의 재생 목록을 사용하지마십시오 . Video Cloud 재생 목록을 사용하면 브라이트코브 비콘 라이브 녹화를 덮어쓰거나 삭제할 수 있습니다.
    3. 재생 목록의콘텐츠 유형 탭에서이벤트탭을 선택합니다.
    4. 콘텐츠 목록에서 이벤트를 찾은 다음 (+) 버튼을 클릭하여 왼쪽 재생 목록에 추가합니다. 이벤트 순서를 앱에 표시하려는 방식으로 배치할 수 있습니다.
    5. 이벤트와 마찬가지로권한 및 일정탭에서 재생 목록 설정 (시간, 장치, 위치 등) 을 조정합니다.
    6. 재생 목록 버튼에 변경 사항 저장버튼을 클릭합니다.
    7. 화면 오른쪽 상단의도구 (렌치) 버튼을 클릭합니다. 그런 다음 캐시를 지웁니다.

    이때 이벤트는 왼쪽 이미지에 표시된 것처럼 Brightcove Beacon 앱에 컨테이너로 생성됩니다 (이미지의 이벤트 이름과 사진은 예시용으로만 사용). 이벤트를 클릭하면 오른쪽에 이미지와 비슷한 내용이 표시됩니다. 라이브 비디오 스트리밍을 실제로 시작하려면 Video Cloud Live Platform에서 전송을 설정해야 합니다.

    Studio에서 라이브 이벤트 만들기

    다음 단계에 따라 브라이트코브 비콘으로 스트리밍될 실제 라이브 이벤트를 만듭니다.

    1. 비디오 클라우드 스튜디오에 로그인하고 탐색 헤더에서Live를클릭합니다.
    2. 이벤트 관리 페이지에서 이벤트생성을누릅니다. 라이브 이벤트 생성페이지가 열립니다.
    3. 이벤트 상세내역을 입력합니다.
      • 이벤트 이름 - 이벤트 이름입니다. 라이브 이벤트가 비디오 에셋으로 저장된 경우 (최대 255자) 미디어 모듈에도 이 이름이 표시됩니다.
      • 간단한 설명 - 이벤트 설명 (최대 250자)
      • 태그 - 이벤트와 연결할 태그입니다. 라이브 이벤트가 비디오 에셋으로 저장되면 태그가 비디오에 할당됩니다 (최대 128자 및 1200 태그).
      • 참조 ID - 이벤트의 참조 ID입니다. 고유해야 합니다 (최대 150자).
    4. 라이브 이벤트를 소셜 미디어 플랫폼으로 스트리밍하려면허용을클릭하고, 그렇지 않으면허용되지 않음을클릭합니다. 이것은 할 수 있습니다이벤트 스트리밍이 시작되면또는 사용하여소셜 모듈 .
    5. 라이브 인제스트 프로파일을선택합니다. 드롭다운 목록에 모든 라이브 인제스트 프로필이 나열됩니다. 인제스트 프로파일은 작성되는 라이브 변환의 품질과 수를 제어합니다. 프로파일 관리링크를 클릭하면 다른 브라우저 탭에서 인제스트 프로파일 편집기가 열립니다. 인제스트 프로파일 편집기에 액세스하려면 계정 관리자여야 합니다. 선택한 인제스트 프로파일을 사용하여 작성될 변환을 보려면 RENDITION스 링크를 클릭합니다.
      세 계층의 비트율 및 청구 세부 정보는 다음과 같습니다.
      • 계층 1은 최대 5,500Kbps를 허용하며 1.0x 시간으로 청구됩니다.
      • 계층 2는 최대 10,000Kbps를 허용하며 1.5x 시간으로 청구됩니다.
      • 계층 3은 최대 20,000Kbps를 허용하며 2.0x 시간으로 청구됩니다.
    6. 완료되면 이벤트를 비디오 자산으로 변환을선택하여 라이브 이벤트의 VOD 버전을 Video Cloud 계정에 저장합니다. 이 옵션을 선택하면 선택한라이브 인제스트 프로파일의최고 품질의 변환이 VOD 비디오의 소스로 사용됩니다. 변환을 만들 때 계정의 기본 인제스트 프로필이 사용됩니다. 라이브 이벤트가 종료되는 즉시 VOD 버전을 사용할 수 없습니다. 수집 프로세스가 완료되면 비디오를 사용할 수 있습니다.
      참고: 반복 이벤트를 사용할 때는 이 옵션이 지원되지 않습니다.
    7. 고급 옵션을클릭하고 고급 옵션을 봅니다. 그 중 하나만주의를 기울여야합니다.
    8. 스트림 상태 알림 활성화확인란을 클릭하고 브라이트코브 비콘 CMS의 알림 URL을 추가합니다 (브라이트코브 지원 담당자가 이 URL을 제공합니다).
    9. 이벤트 만들기를클릭하여 라이브 이벤트를 시작합니다. 인코더에서 사용할스트리밍 끝점 (RTMP URL)및스트림 이름등 라이브 이벤트에 대한 세부 정보가 포함된 제어실 페이지가 열립니다. 스트림을 처리하는 데 약 1 분이 소요됩니다.
      라이브 플랫폼에서 스트리밍을 시작하려면 인코더가 필요합니다. 인코더 사용 방법에 대한 자세한 내용은 다음 설명서를 참조하십시오.

      단계별: 텔레스트림 와이어캐스트와 함께 라이브 모듈 사용

      단계별: 오픈 브로드캐스터 소프트웨어 (OBS) 와 함께 라이브 모듈 사용

    10. 제어실의 [ 테스트 이벤트 ] 패널에서 [ 작업 ID ] 및 [ 비디오 ID ] 를 복사하면 브라이트코브 비콘 플랫폼 설정에서 라이브 이벤트를 구성하려면 둘 다 필요합니다.
    11. 그런 다음 브라이트코브 비콘으로 돌아가서이벤트탭을 클릭합니다. 이전에 만든 이벤트를 선택하고기본 데이터설정에서 Live 이벤트의작업 ID를붙여 넣습니다.
    12. Streams탭으로 이동하여 + 새 스트림 추가 버튼을 클릭하여 새 스트림을만듭니다.
    13. 스트림 비디오 ID필드에 이전에 만든비디오 클라우드 이벤트의 비디오 ID를붙여 넣습니다. 또한 스트림에 대한 다음 정보를 입력하십시오.
      • 비디오를 게시할 수 있는 장치 수를 선택합니다.
      • 드롭다운 메뉴에서스트림 유형을라이브로설정합니다.
      • 페이지를 떠나기 전에이벤트 업데이트버튼을 클릭하십시오.
    14. 화면 오른쪽 상단의도구 (렌치) 버튼을 클릭합니다. 그런 다음 캐시를 지웁니다.
    15. 브라이트코브 비콘 앱을 열고 이벤트를 선택합니다 (이미지의 이벤트 이름과 사진은 예시용으로만 사용).
    16. 재생버튼을 클릭하면 브라이트코브 비콘 앱에서 Video Cloud에서 라이브 이벤트 스트리밍을 볼 수 있습니다.

    이벤트가 종료되면 전체 또는 일부를 Video Cloud의 비디오 에셋이 되는 클립 또는 브라이트코브 비콘에 표시할 수 있는 재생 가능한 스트림으로 저장할 수 있습니다. 이 프로세스는 이 문서의 다음 섹션에서 자세히 설명합니다.

    라이브 이벤트에서 클립 만들기

    나중에 스트리밍하기 위해 비디오를 VOD 녹화 비디오로 유지하려면 브라이트코브 Beacon Live 이벤트 클립을 만들어야 합니다.

    라이브 모듈은 완료된 이벤트에서 비디오 클립을 만드는 기능을 제공합니다. 라이브 이벤트가 진행되는 동안에도 클립을 만들 수 있습니다.

    전체 클리핑 기능 목록을 보려면라이브 모듈을 사용하여 비디오 클립 만들기설명서를 참조하십시오.

    1. 라이브 모듈에서 왼쪽 패널의클립 만들기탭으로 이동하십시오.
    2. 클리핑 버튼 또는 바로 가기 키를 사용하여 클립의 시작점과 끝점을 설정합니다. 당신은 또한 전체 비디오를 유지하기 위해저장버튼을 클릭 할 수 있습니다.
    3. 클립 비디오 속성을 입력합니다.
      • 클립 이름 - 클립의 이름
      • 간단한 설명 - 클립에 대한 설명
      • 태그 - 클립과 연결할 태그
      • 참조 ID - 클립에 할당할 참조 ID
      • 소스 광고 중단 콘텐츠 제거 - 이 옵션은 클립 내에서 트리거된 광고 나누기를 제거하여 프로그램 콘텐츠만 유지합니다.
        참고: 소스 광고 중단 제거 확인란은 스트림이 SSAI를 사용하도록 설정된 경우에만 나타납니다.
    4. 클립 만들기를클릭합니다. 클립의 코드 변환 프로세스가 시작됩니다. 클립은 계정의 기본 인제스트 프로필을 사용하여 코드 변환됩니다.

    재생 가능한 스트림 비디오로 브라이트코브 비콘에 클립 추가

    여기에서 새로 만든 클립 비디오의 ID를 사용하여 브라이트코브 비콘에서 재생할 수 있는 스트림으로 만듭니다.

    1. 비디오 클라우드 스튜디오의 라이브 모듈에 있는 동안 생성된 클립의 번호ID를복사합니다.
    2. 브라이트코브 비콘 플랫폼에서이벤트탭으로 이동하여Streams메뉴를 선택합니다. 다음과 같은 양식이 표시됩니다.
    3. Studio의 라이브 플랫폼에서 복사한ID를비디오 스트림 ID필드에 붙여 넣습니다.
    4. 스트림 유형드롭다운에서Live를스트림으로변경합니다.
    5. 기본 데이터탭을 클릭하고상태를종료에서완료로변경합니다.
    6. 나가기 전에 페이지 하단의이벤트 업데이트버튼을 클릭합니다. 그러면 브라이트코브 비콘 앱에서재생버튼을 사용하여 이벤트 기록을 확인할 수 있습니다.

    (정적 진입점) SEP 라이브 이벤트

    SEP Live Events는 주별 프로그램과 같이 이벤트가 반복해서 시작될 때 활성화/비활성화할 수 있는 반복 이벤트입니다. 이 이벤트는 브라이트코브 비콘 앱의 여러 채널로 전송할 수 있는 모든Live Job ID에대한 이벤트로 사용할 수 있습니다.

    • 반복 이벤트를 만들면 Media 모듈에서 해당 이벤트에 대한 비디오가 자동으로 만들어집니다. 이 비디오는 이벤트가 재발될 때마다 필요하므로 삭제하지 마십시오.
    • 인코더가 시작되기 전에 반복 이벤트를 활성화해야 합니다.

    브라이트코브 비콘에서 SEP (정적 진입점) 라이브 이벤트를 생성하려면 다음 단계를 따르십시오.

    1. 라이브 플랫폼에서새 이벤트를 만듭니다 . 이벤트를만드는 방법에 대한 자세한 내용은 라이브 모듈 문서를 사용하여 라이브 이벤트 만들기 및 관리의 새이벤트만들기 섹션을 참조하십시오.
    2. 고급 옵션을확장하고정적 진입점 (SEP) 을 사용하여 반복 이벤트로 만들기옵션을 사용하도록 설정합니다. SEP에 대한 자세한 내용은되풀이 이벤트 작업섹션 Live 모듈을 사용하여 라이브 이벤트 생성 및 관리문서.
    3. 이미 SEP 이벤트가 있는 경우대기탭을 클릭하면 라이브 제어실에서활성화/비활성화할수 있습니다.
    4. 라이브 작업 ID를복사합니다.
    5. 브라이트코브 비콘 플랫폼으로 돌아가서 반복 이벤트의작업 ID를붙여넣어 이벤트를 구성합니다. 이름및이벤트 시작 일자도입력합니다.
    6. 이벤트 날짜 및 시간은 다음과 같은 형식이어야 합니다.
    7. 이벤트 업데이트버튼을 클릭합니다.
    브라이트코브 비콘 CMS는 라이브 이벤트의상태전환을 자동화하고 있지만 정적 진입 점에서는 동일한 SEP와 연결된 여러 브라이트코브 비콘 이벤트가 있을 수 있습니다. 자동화가 늦게 실행되고 중복되는 문제가 있는 경우Status를수동으로 변경하여 CMS 전환을 조정할 수 있습니다. 예를 들어상태필드의사용 가능에서완료로이동합니다.

    브라이트코브 비콘 플랫폼에서 원하는 모든 라이브 이벤트에 대해 이 프로세스를 수행할 수 있습니다. 모두 동일한 실시간 전송에서 스트리밍되지만 각 전송은 이벤트 설정에서 조정한 날짜와 시간이 다릅니다.
