개요

라이브 이벤트를 애플리케이션으로 스트리밍하려면 일련의 단계에 따라 Beacon Studio Live 계정과 Beacon Classic을 연결해야 합니다.

프로세스의 상위 수준 보기는 다음과 같습니다.

  1. Beacon Classic에서 이벤트와 재생 목록을 만듭니다.
  2. Beacon Studio에서 라이브 모듈에 액세스하여 이벤트를 생성하고 작업 ID를 얻으십시오.
  3. Beacon Classic에서 Beacon Studio의 작업 ID를 사용하여 라이브 이벤트를 등록합니다.
  4. 라이브 비디오는 Beacon 앱에서 스트리밍을 시작합니다.

이 문서의 마지막 섹션에서는 정적 진입점을 사용하여 Live에서 다른 유형의 이벤트를 만드는 단계를 볼 수 있습니다. 이를 통해 활성화 및 비활성화할 수 있고 한 번만 구성해야 하는 반복적인 라이브 이벤트를 허용합니다.

이 문서의 나머지 부분에서는 위의 각 단계에 대한 세부 정보를 제공합니다.

Beacon Classic에서 라이브 이벤트 만들기

Beacon Classic에서 이벤트를 생성하려면 다음 단계를 따르십시오.

활성화된 계정만비콘 클래식 라이브가질 것이다재생 목록그리고이벤트탭 섹션에서 사용할 수 있습니다. 활성화하지 않은 경우 Brightcove 지원에 문의하십시오.
  1. Beacon Classic에서이벤트탭을 눌러 이벤트 설정을 시작합니다.
  2. 클릭새 이벤트 추가 .
  3. 라이브 이벤트 제공이름그리고이벤트 시작일 .
  4. 에서텍스트 데이터탭에서 이벤트 정보를 입력합니다.
  5. 클릭비텍스트 데이터탭.
  6. 이미지 제공초상화 포스터그리고풍경 포스터아래 요구 사항을 충족합니다. 에 대한 이미지를 제공할 필요가 없습니다. 썸네일그리고와이드 배너 .
    영상 요구 사항
    초상화 포스터 포스터 이미지(2:3)
    최저한의 너비: 1000픽셀 키: 563픽셀
    최고 너비: 4000픽셀 키: 2250픽셀
    풍경 포스터 풍경 이미지(16:9)
    최저한의 너비: 1000픽셀 키: 563픽셀
    최고 너비: 4000픽셀 키: 2250픽셀
  7. 클릭권리 및 일정여기에 표시된 대로 탭:
  8. 다음 필드에 값을 추가합니다.
    • 권한 유형 : 권리 유형을 선택하십시오. 광고하는, 신청 또는 무료.
    • 시작 시간 : 이벤트 시작 날짜를 설정합니다.
    • 종료 시간 : 이벤트 종료 날짜를 설정합니다.
    • 최대 스트림 : 라이브 이벤트를 볼 수 있는 최대 스트림 수를 설정합니다. 기본 숫자는 0입니다.
    • 게시할 기기 : 게시하려는 장치를 추가합니다. 딸깍 하는 소리모든 장치 선택/선택 취소사용 가능한 장치 목록을 보려면
    • 허용 위치 : 제한 사항이 입력되지 않은 경우 콘텐츠를 스트리밍할 위치를 선택합니다. 세계.
    • 거부된 위치 : 콘텐츠를 스트리밍하지 않으려는 위치를 선택하십시오.
  9. 클릭새 이벤트 만들기버튼을 클릭하여 이벤트를 생성합니다.

재생 목록에 Beacon Classic 이벤트 추가

이벤트를 생성하면 다음 작업은 이벤트를 재생 목록에 추가하는 것입니다. 다음 단계는 해당 프로세스를 안내합니다.

  1. Beacon Classic에서재생 목록탭.
  2. 클릭새 재생 목록 추가단추.
    라이브 이벤트를 포함할 Beacon Classic에서 재생 목록을 만들어야 합니다. 하지 않다 Video Cloud의 재생 목록을 사용하십시오. Video Cloud 재생 목록을 사용하면 Beacon Classic Live 녹화를 덮어쓰거나 삭제할 수 있습니다.
  3. 에재생 목록 , 콘텐츠 유형 탭에서이벤트탭.
  4. 콘텐츠 목록에서 이벤트를 찾아 (+) 버튼을 클릭하여 왼쪽 재생 목록에 추가합니다. 앱에 표시할 이벤트의 순서를 지정할 수 있습니다.
  5. 재생 목록의 설정(시간, 장치, 위치 등)을 조정하십시오. 권리 및 일정탭을 클릭합니다.
  6. 클릭재생 목록에 변경 사항 저장단추.
  7. 클릭도구 (렌치) 버튼은 화면 오른쪽 상단에 있습니다. 그런 다음 캐시를 지웁니다.

이 시점에서 이벤트는 왼쪽 이미지와 같이 Brightcove Beacon 앱에서 컨테이너로 생성됩니다(이미지의 이벤트 이름과 사진은 예시용임). 이벤트를 클릭하면 다음과 유사한 항목이 표시됩니다. 오른쪽에 있는 이미지. 실제로 라이브 비디오 스트리밍을 시작하려면 Beacon Studio 라이브 플랫폼에서 전송을 설정해야 합니다.

Beacon Studio에서 라이브 이벤트 만들기

다음 단계에 따라 Brightcove Beacon 앱으로 스트리밍할 실제 라이브 이벤트를 만드십시오.

모든 Live 모듈 구성 요소에 대한 자세한 내용은 다음을 방문하십시오. 라이브 이벤트 문서 관리.
  1. Beacon Studio에 로그인하고 클릭살다기본 탐색에서
  2. 로부터이벤트 관리페이지, 클릭이벤트 생성 . NS 라이브 이벤트 만들기페이지가 열립니다.
  3. 이벤트 세부 정보를 입력합니다.
    • 이벤트 이름 - 이벤트 이름 이 이름은 라이브 이벤트가 비디오 자산(최대 255자)으로 저장된 경우 미디어 모듈에도 나타납니다.
    • 간단한 설명 - 이벤트 설명(최대 250자)
    • 태그 - 이벤트와 연결할 태그 라이브 이벤트가 비디오 자산으로 저장된 경우 태그가 비디오에 할당됩니다(최대 128자 및 1200 태그).
    • 참조 ID - 이벤트에 대한 참조 ID 고유해야 합니다(최대 150자).
  4. 딸깍 하는 소리허용 된라이브 이벤트를 소셜 미디어 플랫폼으로 스트리밍하려면 다음을 클릭하십시오. 허용되지 않음 . 이것은 할 수 있습니다이벤트 스트리밍이 시작되면또는 다음을 사용하여소셜 모듈 .
  5. 선택라이브 인제스트 프로필 . 드롭다운 목록에는 모든 라이브 수집 프로필이 나열됩니다. 수집 프로필은 생성되는 라이브 변환의 품질과 수를 제어합니다. 클릭하면프로필 관리링크는 다른 브라우저 탭에서 수집 프로필 편집기를 엽니다. 수집 프로필 편집기에 액세스하려면 계정 관리자여야 합니다. 클릭변환링크를 클릭하면 선택한 수집 프로필을 사용하여 생성될 변환을 볼 수 있습니다.
    세 계층의 비트 전송률 및 청구 세부 정보는 다음과 같습니다.
    • Tier 1은 최대 5,500Kbps를 허용하고 1.0x 시간으로 청구됩니다.
    • Tier 2는 최대 10,000Kbps를 허용하고 시간당 1.5배 요금이 부과됩니다.
    • Tier 3는 최대 20,000Kbps를 허용하고 2.0x 시간으로 청구됩니다.
  6. 선택하다완료되면 이벤트를 비디오 자산으로 변환라이브 이벤트의 VOD 버전을 Beacon Studio 계정에 저장합니다. 이 옵션을 선택하면 선택한라이브 인제스트 프로필 VOD 영상의 소스로 사용됩니다. 계정의 기본 수집 프로필은 변환을 만들 때 사용됩니다. VOD 버전은 라이브 이벤트가 종료되는 즉시 사용할 수 없으며 처리 프로세스가 완료되면 비디오를 사용할 수 있습니다.
    메모: 이 옵션은 반복 이벤트를 사용할 때 지원되지 않습니다.
  7. 딸깍 하는 소리고급 옵션고급 옵션을 봅니다. 그 중 하나만 주의를 기울여야 합니다.
  8. 에서 클릭스트림 상태 알림 활성화확인란을 선택하고 Beacon Classic CMS의 알림 URL을 추가합니다(Brightcove 지원 담당자가 이 URL을 제공합니다).
    NS 알림 URL이벤트에서 추가 작업을 트리거하도록 리스너를 설정합니다. VOD 및 라이브 이벤트의 상태 알림 및 상태 변경에 대한 자세한 내용은라이브 API: 알림 문서 .
  9. 딸깍 하는 소리이벤트 생성라이브 이벤트를 시작합니다. 제어실 페이지가 열리고 다음을 포함한 라이브 이벤트에 대한 세부 정보가 표시됩니다. 스트리밍 엔드포인트(RTMP URL)그리고스트림 이름인코더에서 사용됩니다. 스트림을 처리하는 데 약 1분이 소요됩니다.
    라이브 플랫폼에서 스트리밍을 시작하려면 인코더가 필요합니다. 인코더 사용 방법에 대한 정보는 다음 문서를 참조하십시오.

    단계별: Telestream Wirecast와 함께 라이브 모듈 사용

    단계별: OBS(Open Broadcaster Software)와 함께 라이브 모듈 사용

  10. 에서제어실 , 복사작업 ID그리고동영상 ID ~로부터테스트 이벤트패널에서 Beacon Classic 플랫폼 설정에서 라이브 이벤트를 구성하려면 둘 다 필요합니다.
  11. 다음으로 Beacon Classic으로 돌아가서이벤트탭. 이전에 생성한 이벤트를 선택하고기본적인 정보설정 붙여넣기 작업 ID 당신의 라이브 이벤트의.
  12. 로 이동스트림탭을 클릭하고 새 스트림을 생성합니다. + 새 스트림 추가단추.
  13. 에서스트림 비디오 ID필드, 붙여넣기동영상 ID이전에 생성한 Beacon Studio 이벤트에서. 또한 스트림에 대한 다음 정보를 입력합니다.
    • 비디오를 게시할 수 있는 장치의 수를 선택하십시오.
    • 설정스트림 유형에게살다드롭다운 메뉴에서.
    • 그리고업데이트 이벤트버튼을 눌러 페이지를 떠나십시오.
  14. 클릭도구 (렌치) 버튼은 화면 오른쪽 상단에 있습니다. 그런 다음 캐시를 지웁니다.
  15. 비콘 앱을 열고 이벤트를 선택합니다(이미지의 이벤트 이름과 사진은 예시일 뿐입니다).
  16. 클릭놀다버튼을 누르면 이제 Beacon Classic 앱의 Beacon Studio에서 스트리밍되는 라이브 이벤트를 볼 수 있습니다.

이벤트가 끝나면 전체 또는 일부를 Beacon Studio에서 비디오 자산이 되는 클립으로 저장하거나 Beacon Classic에서 표시할 수 있는 재생 가능한 스트림으로 저장할 수 있습니다. 프로세스는 이 문서의 다음 섹션에 자세히 설명되어 있습니다.

라이브 이벤트에서 클립 만들기

비디오를 나중에 스트리밍할 수 있도록 VOD 녹화 비디오로 유지하려면 Beacon Classic Live 이벤트의 클립을 생성해야 합니다.

라이브 모듈은 완료된 이벤트에서 비디오 클립을 만드는 기능을 제공합니다. 라이브 이벤트가 진행되는 동안에도 클립을 만들 수 있습니다.

전체 클리핑 기능 목록을 보려면 다음을 검토하십시오. 라이브 모듈을 사용하여 비디오 클립 만들기선적 서류 비치.

  1. 라이브 모듈에서클립 만들기탭을 클릭합니다.
  2. 클리핑 버튼 또는 바로 가기 키를 사용하여 클립의 시작 및 끝점을 설정합니다. 당신은 또한구하다버튼을 눌러 전체 비디오를 유지합니다.
  3. 클립 비디오 속성을 입력합니다.
    • 클립 이름 - 클립의 이름
    • 간단한 설명 - 클립에 대한 설명
    • 태그 - 클립과 연결할 태그
    • 참조 ID - 클립에 할당할 참조 ID
    • 소스 광고 시간 콘텐츠 제거 - 이 옵션은 클립 내에서 실행된 광고 시간을 제거하고 프로그램 콘텐츠만 유지합니다.
      메모: 소스 광고 시간 제거 확인란은 스트림이 SSAI를 활성화한 경우에만 나타납니다.
  4. 딸깍 하는 소리클립 만들기 . 클립의 트랜스코딩 프로세스가 시작됩니다. 클립은 계정의 기본 수집 프로필을 사용하여 코드 변환됩니다.

재생 가능한 스트림 비디오로 Beacon Classic에 클립 추가

여기에서 새로 만든 클립 비디오의 ID를 사용하여 Beacon Classic에서 재생할 수 있는 스트림으로 만듭니다.

  1. Beacon Studio의 Live 모듈에 있는 동안 번호를 복사합니다. ID당신이 만든 클립의.
  2. Beacon Classic 플랫폼에서이벤트탭을 선택하고스트림메뉴. 다음과 같은 형식이 표시됩니다.
  3. 붙여넣기 ID방금 Studio의 Live 플랫폼에서 복사했습니다. 스트림 비디오 ID들.
  4. 로부터스트림 유형드롭다운 변경살다에게개울 .
  5. 에서 클릭기본적인 정보탭하고 변경상태 ~에서종료에게완성 된 .
  6. 떠나기 전에업데이트 이벤트버튼은 페이지 하단에 있습니다. 이렇게 하면놀다이벤트 녹화를 보려면 Beacon 앱에서 버튼을 사용할 수 있습니다.

(정적 진입점) SEP 라이브 이벤트

SEP 라이브 이벤트는 예를 들어 주간 프로그램과 같이 이벤트가 계속해서 시작될 때 활성화/비활성화할 수 있는 반복 이벤트입니다. 이 이벤트는 모두를 위한 하나로 사용할 수 있습니다. 라이브 작업 ID Beacon App에서 여러 채널로 전송할 수 있습니다.

  • 반복 이벤트를 생성하면 미디어 모듈에서 자동으로 해당 이벤트에 대한 비디오가 생성됩니다. 이벤트가 반복될 때마다 필요하므로 이 비디오를 삭제하지 마십시오.
  • 인코더가 시작되기 전에 반복 이벤트가 활성화되어야 합니다.

Beacon Classic에서 SEP(정적 진입점) 라이브 이벤트를 생성하려면 다음 단계를 따르십시오.

  1. 라이브 플랫폼에서는새 이벤트 만들기 . 이벤트를 만드는 방법에 대한 자세한 내용은 다음을 참조하십시오. 새 이벤트 만들기섹션 라이브 모듈을 사용하여 라이브 이벤트 생성 및 관리 문서.
  2. 확장고급 옵션 , 활성화 SEP(정적 진입점)를 사용하여 반복 이벤트로 생성옵션. SEP에 대한 자세한 내용은되풀이 이벤트 작업섹션 라이브 모듈을 사용하여 라이브 이벤트 생성 및 관리 문서 .
  3. 이미 SEP 이벤트가 있는 경우대기탭을 클릭하면활성화/비활성화라이브 관제실에서.
  4. 복사라이브 작업 ID .
  5. Beacon Classic 플랫폼으로 돌아가서 이벤트를 구성하십시오. 작업 ID반복되는 이벤트의. 또한 공급이름그리고이벤트 시작일 .
  6. 이벤트 날짜와 시간은 다음 형식이어야 합니다.
  7. 에서 클릭업데이트 이벤트단추.
Beacon Classic CMS는상태라이브 이벤트에 대한 전환이지만 정적 진입점에는 동일한 SEP와 연결된 여러 Beacon Classic 이벤트가 있을 수 있습니다. 자동화에 문제가 늦게 실행되고 겹치는 경우 수동으로 변경하여 CMS 전환을 조정할 수 있습니다. 상태 . 예를 들어, 사용 가능에게완성 된에서상태들.

Beacon Classic 플랫폼에서 원하는 모든 라이브 이벤트에 대해 이 프로세스를 수행할 수 있습니다. 그들 모두는 동일한 라이브 전송에서 스트리밍되지만 이벤트 설정에서 조정한 날짜와 시간이 각각 다릅니다.
