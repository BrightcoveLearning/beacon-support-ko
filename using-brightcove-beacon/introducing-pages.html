페이지 카테고리

페이지 유형은 세 가지 범주로 그룹화할 수 있습니다. 각 카테고리 및 세부 정보는 다음과 같습니다.

  1. 구성 불가능
    • 앱에 포함할지 여부를 선택하는 페이지입니다.
    • 문서구성할 수 없는 페이지 포함이러한 유형의 페이지 중 하나를 앱에 추가하는 방법을 포함하여 더 자세한 정보를 제공합니다.
    • 구성할 수 없는 페이지의 예는 다음과 같습니다. 설정그리고가입하기 , 여기에 표시:
  2. 화면
    • 캐러셀, 목록 등에 동영상 콘텐츠가 포함된 페이지입니다.
    • 사용하여 고도로 구성 가능블록 , 레이아웃그리고재생 목록 . 모바일 플랫폼에 대한 두 가지 예가 표시되고 웹 앱 예가 뒤따릅니다.
    • screen-hero_image
      • 특별한 유형의 화면 페이지이지만 비디오 콘텐츠 뒤에 이미지를 허용합니다.
      • 웹(가로) 및 모바일(세로)과 같은 다양한 플랫폼에 대해 가로 및 포스터 방향 이미지를 모두 업로드해야 합니다.
      • NS "영웅"화면 상단의 영역은 덮이지 않으며 구성할 수 없습니다.
  3. 리디렉션
    • 리디렉션 페이지 유형은 웹 앱 전용입니다.
    • 리디렉션 공백새 탭에서 웹 페이지를 엽니다.
    • 리디렉션 자체같은 탭에서 웹 페이지를 엽니다.
    • 에 URL을 지정하십시오. 외부 리디렉션 탭.

화면 페이지 구성

이 문서의 앞부분에서 언급했듯이 화면 페이지에는 비디오가 포함되어 있으며 비디오를 나타내는 이미지 표시 옵션을 제공합니다. 이 섹션에서는 구성 옵션이 앱에서 어떻게 보이는지 스크린샷과 함께 표시됩니다. 화면 페이지에서 가장 높은 수준의 컨테이너인 블록에 대한 정보가 먼저 표시됩니다. 그러면 비디오를 나타내는 이미지가 표시되는 방식을 제어하는 레이아웃 정보가 표시됩니다.

블록

노란색으로 강조 표시된 블록 유형은 다음과 같습니다.

다음은 각 유형의 이미지입니다.

단일 블록

NS 단일 블록 하나의 레이아웃/재생 목록 조합만 블록에 추가할 수 있습니다. 스크린샷은 다음을 보여줍니다 단일 블록 회전 목마를 표시합니다. 여러 레이아웃/재생 목록 조합을 추가하려는 경우 단일 블록, 또는 단일 블록 여러 레이아웃/재생 목록 조합이 포함된 경우 오류가 발생합니다.

목록 차단

NS 목록 차단 블록에 여러 레이아웃/재생 목록 조합을 추가할 수 있습니다. 스크린샷은 다음을 보여줍니다 목록 차단 3개의 캐러셀이 포함되어 있습니다.

tabed_block

NS 탭 블록 여러 레이아웃/재생 목록 조합을 블록에 추가할 수 있지만 한 번에 하나만 표시되고 탭을 통해 사용자는 표시할 항목을 선택할 수 있습니다. 두 가지 제한 사항이 적용됩니다.

  • tabed-block 내부의 레이아웃은 동일해야 합니다.
  • 탭 블록 내에서 레이아웃은 그리드여야 합니다.
스크린샷은 다음을 보여줍니다 탭 블록 재생 목록을 선택할 수 있습니다.

드롭다운 블록

NS 드롭다운 블록 여러 레이아웃/재생 목록 조합을 블록에 추가할 수 있지만 한 번에 하나만 표시되며 드롭다운을 통해 사용자가 표시할 항목을 선택할 수 있습니다. 두 가지 제한 사항이 적용됩니다.

  • 드롭다운 블록 내부의 레이아웃은 동일해야 합니다.
  • 드롭다운 블록 내에서 레이아웃은 그리드여야 합니다.
스크린샷은 다음을 보여줍니다 드롭다운 블록 (노란색 사각형 안) 세 개의 재생 목록을 선택할 수 있습니다.

레이아웃

선택한 레이아웃 위의 모든 블록 예에서 회전 목마가 있었습니다. 캐러셀은 각 동영상의 이미지를 가로 목록으로 표시합니다. 선택할 수있는 유일한 레이아웃은 아닙니다. 사용 가능한 레이아웃은 노란색으로 강조 표시됩니다.

각 레이아웃 유형의 스크린샷 대신 각 레이아웃 특성의 예가 표시됩니다. 위의 노란색으로 강조 표시된 유형은 서로 다른 특성의 조합입니다.

캐러셀 및 라벨

캐러셀은 이미지를 가로 행으로 표시하고 레이블은 이미지의 작은 하단 부분을 오버레이합니다.

라벨 아웃

라벨은 이미지 아래에 있습니다.

추천

이미지가 더 큽니다(빨간색 상자, 그 아래 일반 캐러셀). 항상 그런 것은 아니지만 종종 추천 레이아웃이 페이지 상단에 사용됩니다.

풍경

이미지는 가로 비율입니다.

포스터

이미지는 포스터(세로) 비율입니다.

그리드 3 및 그리드 4

두 가지 유형의 그리드는 2차원 행/열 패턴으로 이미지를 표시합니다. 그리드 다음의 숫자 3 또는 4는 이미지의 상대적 크기를 나타냅니다. 항상 표시될 열의 수를 나타내지는 않습니다. 이 스크린샷은 너비가 800px인 브라우저를 보여주며 Grid 3에는 3개의 이미지로 구성된 2행이 있습니다. 그리드 4는 이미지가 더 작기 때문에 가로질러 4개의 이미지를 표시합니다.

이 다음 스크린샷은 여전히 Grid 3과 Grid 4를 모두 사용하고 있지만 브라우저 너비는 1600px입니다. 이제 Grid 3 이미지의 가로가 4개이고 Grid 4 레이아웃의 5개 이미지가 화면 전체에 맞을 수 있습니다. 다시 Grid 3 이미지가 더 큰 것을 볼 수 있습니다.

이것으로 블록 및 레이아웃 옵션을 마칩니다.

공통 페이지 구성 설정

새 페이지를 만들거나 기존 페이지를 편집할 때 Brightcove Beacon에서 다음 탭이 표시됩니다.

NS 기본적인 정보탭은 페이지 유형을 선택하고 구성하는 곳입니다. 화면페이지. 다른 탭은 모든 페이지 유형에 대해 동일합니다. NS 공통 구성 설정문서 세부 사항에 필요한 정보텍스트 데이터 , 비텍스트 데이터 , 태그그리고권리 및 일정탭.
